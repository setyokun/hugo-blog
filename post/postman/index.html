Cara Testing Rest API dengan Postman

Jul 19, 2019

Kali ini kita akan membahas bagaimana testing Rest API dengan Postman. Jadi ada 4 method yang sering digunakan dalam testing API yaitu GET, POST, PUT, dan DELETE. Pertama-tama kita buka Postman dan masukkan endpoint beserta list API yang sudah tersedia di https://reqres.in/ ini. Sung mari kita coba :

1 . GET
Pada method ini tuh untuk mengambil data. Ganti Fokus ke Postman, ganti methodnya seperti nomor (1) di bawah ini :

Selanjutnya copy URL https://reqres.in/api/users/2 ini dan masukkan ke Postman seperti pada nomor (2). Kemudian Klik Send seperti pada nomor (3) dan validasi response dari hasil requestnya pada nomor (4) dan (5). Pastikan response pada nomor (4) dan (5) seperti response yang ada pada web https://reqres.in/ di bawah ini nomor (3) dan (4) :

2 . POST
Pada method ini tuh untuk menambahkan data. Kemudian ganti methodnya seperti nomor (1) di bawah ini :

Selanjutnya copy URL https://reqres.in/api/users ini dan masukkan ke Postman seperti pada nomor (2). Trus klik body (nomor 3) >> raw (nomor 4) >> JSON (nomor 5) >> masukkan request bodynya juga (nomor 6)

{
    "name": "morpheus",
    "job": "leader"
}

Lanjut di Send (nomor 7). Dan akan mendapat response seperti gambar :

Pastikan Response pada nomor 1 dan 2 diatas sama seperti response yang ada pada web https://reqres.in/ di bawah ini pada nomor 3 dan 4 :

3 . PUT
Pada method ini tuh untuk mengupdate data. Kemudian ganti methodnya seperti nomor (1) di bawah ini :

Selanjutnya copy URL https://reqres.in/api/users/2 ini dan masukkan ke Postman seperti pada nomor (2). Trus klik body (nomor 3) >> x-www-form-urlencoded (nomor 4) >> masukkan key dan valuenya (nomor 5) seperti gambar diatas.

Trus klik send (nomor 6) dan validasi response dari nomor 7 dan 8. Pastikan Responsenya sama seperti response yang ada pada web https://reqres.in/ di bawah ini pada nomor 3 dan 4 :
