Cara Mudah Membuat Website Statis dengan Hugo dan Github

Jul 10, 2019

1 . Download dan Install Hugo versi terbaru disini Hugo Release.

2 . Gunakan perintah hugo new site <direktori> untuk membuat site baru yang akan digenerate oleh hugo

3 . Buat Repositori di github dengan nama blog

4 . Kemudian clone repo github blog tadi ke lokal dengan perintah git clone <urlRepo>

5 . Copykan file-file yang tadi digenerate (step 2) oleh hugo kedalam direktori hasil clone dari repo github blog, jadi sekarang di direktori blog ada file-file generate hugo dan .git dari hasil clone

6 . Tambahkan theme sesuai dengan keinginanmu, kamu bisa mencarinya disini Hugo Themes.

7 . Pilih salah satu theme, kemudian clone dengan perintah git clone <urlRepoTheme> di folder themes

8 . Copy file config.toml dari direktori themes yang dipilih tadi, disini tempatnya themes >> nama theme >> exampleSite >> config.toml ke root direktori

9 . Jalankan perintah hugo server untuk menjalankan website statis yang digenerate oleh hugo dan dapat diakses di http://localhost:1313/

10 . Lihat hasil dari website+layout yang ada, kamu bisa mengaturnya di config.toml yang berada di root direktori

11 . Gunakan ctrl + C untuk menghentikan server hugo

12 . Push semua file yang ada di repo lokal kita ke repo github blog dengan cara >> git add . >> git commit -m "message" >> git push origin master.

13 . Buat Repositori di github dengan nama username.github.io

14 . Ketikkan perintah git submodule add -b master <urlSite.github.io> public untuk menambahkan/menanamkan repository baru pada repository inti di direktori public, nantinya pada direktori public tersrbut akan ada .git

15 . Pastikan semua berjalan lancar dan sesuai keinginan, Kemudian Langsung lanjut dengan perintah hugo untuk mengenerate file-file di direktori public

16 . Push to github repo ‘usernmae.github.io’ dengan cara >> cd public >> git add . >> git commit -m "your message" >> git push origin master

17 . Tunggu beberapa menit, kemudian buka github-pages (username.github.io) yang kamu buat tadi daaaaan TADAAA!!!, udah jadi deh :)

18 . Agar lebih mudah untuk post content kamu perlu menambahkan deploy.sh , yang didalamnya terdapat file seperti dibawah ini :

#!/bin/bash

echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"

# Build the project.
hugo # if using a theme, replace with `hugo -t <YOURTHEME>`

# Go To Public folder
cd public
# Add changes to git.
git add .

# Commit changes.
msg="rebuilding site `date`"
if [ $# -eq 1 ]
  then msg="$1"
fi
git commit -m "$msg"

# Push source and build repos.
git push origin master

# Come Back up to the Project Root
cd ..

20 . Jangan lupa kasih akses pada file tersebut dengan cara chmod +x deploy.sh, lalu untuk cara penggunaanya adalah dengan cara ./deploy.sh "your message". simple kan :)

NB :
- Jangan hapus file2 didalam public, nanti hugo akan meng-overide / replace file2 tersebut kecuali file .git
- kalo file .git pada direktori public terhapus, buat lagi dengan git submodule sama seperti step 14, dengan syntax git submodule add -b master --force <urlSite.github.io> public
